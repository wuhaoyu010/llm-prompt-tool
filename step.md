# 大模型提示词管理与测试工具开发步骤

## 第一阶段：项目初始化与后端搭建

1.  **创建项目结构**:
    -   在工作区创建 `app` 文件夹。
    -   在 `app` 内创建以下结构:
        -   `main.py`: Flask 应用主文件。
        -   `database.py`: 数据库模型和初始化脚本。
        -   `static/`: 存放 CSS 和 JavaScript 文件。
        -   `templates/`: 存放 HTML 模板文件。
        -   `uploads/`: 存放用户上传的图片。

2.  **设置后端环境**:
    -   使用 Python Flask 作为后端框架。
    -   使用 SQLite 作为数据库，通过 SQLAlchemy 进行 ORM 操作。

3.  **数据库模型设计 (`database.py`)**:
    -   `Defect`: 存储缺陷信息 (如 `hand_phone`, `玩手机`, `作业期间使用手机` 等)。
    -   `PromptVersion`: 存储每个缺陷的历史提示词版本，与 `Defect` 关联。
    -   `TestImage`: 存储上传的测试图片信息（路径、所属缺陷），与 `Defect` 关联。
    -   `BoundingBox`: 存储每张图片上的标注框（原始坐标、归一化坐标），与 `TestImage` 关联。
    -   `TestResult`: 存储测试结果，关联 `PromptVersion` 和 `TestImage`。

## 第二阶段：核心后端功能开发

1.  **实现数据库操作**:
    -   在 `database.py` 中编写初始化数据库的函数。
    -   在 `main.py` 中提供用于创建、查询 `Defect`, `PromptVersion`, `TestImage` 等的辅助函数。

2.  **开发 API 端点 (`main.py`)**:
    -   `/`: 渲染主页面。
    -   `POST /api/defect`: 创建/更新一个缺陷的定义。
    -   `GET /api/defects`: 获取所有缺陷的列表。
    -   `GET /api/defect/<defect_id>`: 获取特定缺陷的详细信息，包括其最新的提示词和所有关联的测试图片。
    -   `POST /api/prompt`: 为某个缺陷保存一个新的提示词版本。
    -   `POST /api/image/upload`: 上传图片，并将其与一个缺陷关联。
    -   `POST /api/test`: 接收图片ID和用户绘制的标注框，执行单次测试。
        -   标准化坐标。
        -   格式化提示词。
        -   调用大模型 API (初期使用模拟函数)。
        -   保存并返回结果。
    -   `POST /api/regression_test`: 接收一个提示词版本ID，对该缺陷下的所有历史图片进行回归测试。

## 第三阶段：前端开发

1.  **主页面 (`templates/index.html`)**:
    -   基于 `code.html` 和 `screen.png` 设计布局。
    -   左侧为缺陷列表和版本历史。
    -   右侧为主要工作区，包含提示词编辑器、图片上传和标注区、测试结果展示区。

2.  **图片标注功能 (`static/main.js`)**:
    -   实现图片上传预览。
    -   使用 `<canvas>` 元素承载图片，并监听鼠标事件 (`mousedown`, `mousemove`, `mouseup`) 来绘制矩形框。
    -   在绘制完成后，计算框相对于图片原始尺寸的坐标。
    -   将坐标发送到后端前，进行归一化处理（缩放到 1000x1000）。

3.  **前后端交互 (`static/main.js`)**:
    -   使用 `fetch` API 调用后端接口。
    -   动态加载缺陷列表和提示词历史。
    -   点击“保存提示词”时，将编辑器内容发送到后端。
    -   上传图片并绘制标注框后，点击“开始测试”，将图片ID和框坐标发送到 `/api/test`。
    -   实现“一键回归测试”按钮，调用 `/api/regression_test` 并展示所有结果。
    -   动态渲染测试结果。

## 第四阶段：整合与测试

1.  **模拟大模型 API**:
    -   在后端创建一个 `call_llm_api` 函数，该函数接收格式化后的提示词，并返回一个符合格式要求的模拟 JSON 响应。

2.  **端到端测试**:
    -   完整测试流程：创建缺陷 -> 保存提示词 -> 上传图片 -> 标注 -> 单次测试。
    -   保存新版提示词 -> 执行回归测试。
    -   检查数据库中数据是否正确保存。

3.  **代码完善与部署**:
    -   清理代码，添加必要的注释。
    -   将所有开发完成的代码整理到 `app` 文件夹下。
